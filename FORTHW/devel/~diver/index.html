"базовая" форт-система: SP-Forth

Основные интересы в области Форта - применение форт-технологий в практике научного эксперимента (расчеты, системы управления и обработки реального времени), создание кросскомпиляторов для микроконтроллеров и DSP, разработка mini FORTH OS, реализация форта в железе. -- MATRIX - набор библиотек для работы с матрицами ; (в стиле matrix.f Дмитрия Якимова)

состав:

  • matrix2_1.f -версия библиотеки matrix.f Дмитрия Якимова, обновленная и  исправленная
  • matrix_3d.f -работа с трехмерными массивами(элементы массива - двумерные матрицы), версия 1.2
  • matrix_ext.f -слова организации для интерфейса со словами библиотек из FSL(копирование строки|столбца матрицы в|из динамический массив)
  • matrix_dsp.f -цифровая обработка матриц(на данный момент реализованы преобразования Фурье по столбам, строкам и двумерное(из FSL), пока только для флоатов)

возможности:

- работа в многозадачном режиме (все критические внутренние структуры USER..);

- работа с различными типами данных (байты, слова, целые, числа с плавающей точкой), достигается применением векторных слов, заменяемых при переключении формата. (для этого пришлось слегка изменить некоторые слова matrix.f версии 1.2, но все имена и сематника слов сохранилась);

- matrix2_1.f добавлены некоторые операции алгебры матриц слова: Транспонировать, СделатьТреугольной, ОпределительМатр, Минор, ПрисоединеннаяМатр, ОбратнаяМатр, и Инициализировать_из_файла; добавлены слова Матр-, Число/Матр.

исправления: matrix2_1.f версия 2.23 - исправлен откровенный bug в слове Матр+ и некоторых других - некорректное использование стека возвратов внутри цикла.

!!! от использования USER переменных и векторных слов определенных по USER-VECT пришлось отказаться из-за слишком больших заморочек работы с ними в Windows GUI программах.

- matrix_3d.f версия 1.2 - добавлены слова: ПодМассив - подсчет объема памяти для массива, ОбъявитьМассивом - создает структуру массива в уже выделенной области памяти(эти слова добавлены в основном для исправления глюка, моего или Винды при выделении памяти под массив в GUI программах (использовались win-либы на основе joop Дмитрия Якимова)), ВычислитьСреднее, ЗаписатьФайл.

matrix_3d.f версия 1.2 исправлены некоторые баги при определении массивов, работы со слоями.

!!! недостатки:

- универсальность универсальностью, но скорость по-моему упала раза в 2-3 по моим оценкам по сравнению с исходной версией ;

- наиболее серьезный из замеченных в matrix2_1.f - слово Инициализировать_из_файла работает не корректно - до сих пор не исправлен;

-- comm.f работа с последовательным портом под Windows - тестированная и вполне рабочая версия

-- within.f версия слова within - выдает корректный результат и при случае некорректного задания входных параметров, см. результаты тестирования в файле.

-- ring.f реализация кольцевой многозадачности для spf4. – в процессе переработки, старая(рабочая версия после первого EOF).

возможности:

- реализован кольцевой принцип переключения задач со всеми его плюсами и минусами. задачи работают в общей области памяти, но каждая имеет свои стеки и область памяти, именуемой каналом (слова для работы с ним не определены). скорость переключения задач примерно 2 млн/сек. ( WinME на Pentium 166 )

- добавлены приоритеты: 0 - макс., чем х больше, тем больше задача пропускает оборотов кольца, т.е. приоритет меньше, -1 - задача спит и будет разбужена другой задачей.

недостаки:

вообще их куча, основные:

- портабельности никакой, даже на spf2 не идет, почему-то ???

- переменные общие, никаких USER для задачи в рамках этой модели пока нет, другая задача их видит и использует.

- в многозадачном режиме не работает (в потоках windows созданных по TASK: )

- память по завершении не освобождается, не до этого было

- плюс все остальные :-)
