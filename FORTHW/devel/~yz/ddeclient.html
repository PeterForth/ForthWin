ddeclient.f  Клиент DDE

DDE - это протокол связи между программами. Фирмой Microsoft он объявлен устаревшим и заменен более мощным (и более сложным) OLE automation. Хотя на практике для небольших приложений, обменивающихся небольшим количеством данных, DDE до сих пор вполне адекватен и удобен. Типичная область применения - управление другими программами. Библиотека ddeclient определяет слова, необходимые для доступа к серверам DDE.

dde-init ( -- 0/err)

Инициализирует подсистему DDE. В случае ошибки возвращается ее код, иначе - 0.

dde-destroy ( -- )

Выгружает подсистему DDE.

dde-connect ( zserver ztopic -- hconv 0 / 0 0 / err)

Пытается подсоединиться к серверу DDE и запросить у него указанную тему. Если подсоединение прошло успешно, возвращается дескриптор соединения hconv. Если дескриптор равен нулю, то сервер не найден. Если возникла ошибка, возвращается ее код.

dde-disconnect ( hconv -- 0 / err)

Отключает указанное соединение и освобождает сервер.

dde-execute ( zcommand hconv -- 0 / err)

Требует от сервера выполнения указанной команды.

dde-request ( z hconv -- z1 0 / 0 0 / err)

Запрашивает информацию у сервера. Строка, которую вернул сервер, выделяется из памяти, поэтому после использования ее надо освободить вызовом FREE. Если вместо строки возвращается 0, это означает, что в данный момент сервер занят и запрос надо через какое-то время повторить.
